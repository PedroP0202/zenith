# Definição Core: Zenith Streaks

## 1. O que é um Streak?
Um streak (ofensiva/sequência) é a **representação numérica da consistência ininterrupta**. É a contagem de dias seguidos em que o utilizador cumpriu com sucesso o hábito ou tarefa a que se comprometeu.

## 2. O que acontece quando falha?
Cai para **zero**. Sem exceções.
Nesta versão, não existem "streak freezes", "vidas extra" ou tolerâncias. Quebrar a sequência significa perder todo o progresso acumulado na contagem, devolvendo o utilizador ao dia zero. É esta consequência dura que dá valor real ao streak.

## 3. Frequências permitidas
**Diária**. 
Para simplificar e criar o máximo de clareza, a única frequência permitida no sistema atual é a diária. O compromisso deve ser realizado todos os dias.

## 4. O que é o Sucesso Diário?
O sucesso diário é **binário (Feito / Não Feito)**. 
Significa que o utilizador abriu a aplicação e marcou a sua tarefa como concluída até à meia-noite (ou fim do ciclo diário definido) daquele mesmo dia. Não há métricas de qualidade, tempo ou volume (ex: não importa se correu 10km ou 1km, importa se assinalou "Fui correr").

---

## O que NÃO vamos fazer AGORA (Anti-escopo)
- **Frequências Complexas**: "3 vezes por semana", "apenas dias úteis", "dia sim, dia não".
- **Gamification de Perdas**: "Streak freezes", "repair tickets", vidas.
- **Temporizadores e Métricas**: Contar minutos gastos, páginas lidas, litros de água. O check é apenas binário.
- **Social**: Adicionar amigos, leaderboards, feeds, partilha de hábitos.
- **Categorização**: Tags, pastas, agrupamento de hábitos por áreas de vida.
